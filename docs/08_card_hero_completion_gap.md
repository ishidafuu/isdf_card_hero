# カードヒーロー完成度向上 差分棚卸し

## 目的

オリジナル要素を入れる前に、通常戦としての完成度を上げる。

このドキュメントでは、現プロトタイプに残るカードヒーローらしさの不足点を、実装順に近い粒度で整理する。

## 優先度A: 先に整える

### 対戦設定

- seed固定で同じデッキと展開を再現できる。
- 先攻/後攻を選べる。
- Player vs CPU と CPU vs CPU を画面から切り替えられる。
- 同じ条件ですぐ再戦できる。

現状:

- seed、先攻、対戦モードを画面上部に追加済み。
- CPU先攻時も先攻側だけ初回ドローなしになる。

### 射程と対象の可視化

- 技やカード選択中、対象候補を盤面上で色分けする。
- 味方対象、敵対象、移動先、召喚先、マスター対象を区別する。
- 右パネルにも選択中の技、射程、候補数を出す。

現状:

- 対象候補の色分けとラベル表示を追加済み。
- 手札モンスター選択時、召喚可能な空き枠を召喚先として表示済み。
- 右パネルに選択中アクションの候補概要を追加済み。

### カード詳細の読みやすさ

- 公式説明文だけでなく、発動条件、対象条件、持続を分けて表示する。
- コスト、反動、射程、持続が技ごとに追える。

現状:

- マジックとモンスター技に、条件/対象/持続の分解表示を追加済み。

## 優先度B: 次に詰める

### 固定デッキ確認/編集

- 現在のランダムデッキをseedと一緒に保存できる。
- 30枚固定、同名3枚制限、前衛/後衛/魔法の枚数を確認できる。
- プレイヤーとCPUのデッキを別々に選べる。

現状:

- Deck Setup UIでPlayer/CPU別にランダム/固定を切り替え可能。
- 固定デッキはカード選択UIで追加/削除でき、必要ならカード名またはIDのテキスト直接編集も使える。
- 30枚、同名3枚、前衛12、後衛6、魔法6の検証を画面表示する。
- 山札順はseedで再現される。

### 公式通常戦との差分表

- 公式ルールと実装ルールの差分を、ルール単位で管理する。
- 採用、未採用、後回し、仕様違いを混ぜない。
- スペシャルカードを入れるかどうかは、この差分表を見て判断する。

現状:

- `docs/10_official_battle_gap_table.md` に通常戦、マスター差分、スペシャルカード方針を整理済み。

### CPU判断の説明力

- CPUが選んだ理由だけでなく、強そうな手を選ばなかった理由も追える。
- マジック温存、HPドロー、移動、シールド判断の説明を強化する。
- CPU評価値のデバッグ表示は通常UIとは分ける。

現状:

- CPU判断ログに、上位候補を何点差で見送ったかを短く追加済み。
- CPU Decision Historyで、選択理由と見送り候補をログ一覧とは別に確認できる。

### 手動プレイ時の説明力

- 攻撃不可、対象なし、Stone不足、準備中などの理由を、カード情報と行動ボタンから追える。
- ドラッグ操作では、対象を先に指定した攻撃/マジック/移動の確定内容が分かる。
- 勝敗時に、長期戦や山札切れが起きたかを結果表示から確認できる。

現状:

- 右パネルに操作理由パネルを追加済み。
- 勝敗表示にターン数、山札残、HP、山札切れ/長期戦ラベルを追加済み。

## 優先度C: 後段

### マスター差分

- ホワイトマスター以外の導入可否を決める。
- マスター特技、コスト、常時シールド、初期HPの差分を整理する。

### スペシャルカード

- 導入する。ただし通常カード126枚とは別プールにする。
- デフォルトのランダムデッキには混ぜない。
- Deck Setupで明示的に許可した場合だけ固定デッキに投入できる。
- 通常カードと同じ粒度で効果、表示、テストを用意する。
- 導入仕様は `docs/11_special_cards_spec.md` に置く。

### 開発設計

- `rules.ts` は大きくなっているため、今後の大きな効果追加前に分割を検討する。
- 分割設計は `docs/09_rules_split_design.md` に追加済み。
- `rules.ts` を公開ファサードとして残し、内部実装を `src/game/ruleEngine/` へ段階移行する。
- Step 0として定数/盤面順は `src/game/ruleEngine/constants.ts` へ分割済み。
- Step 1としてログ/乱数/cloneは `src/game/ruleEngine/` へ分割済み。
- Step 2として座標、距離、射程判定の純粋処理は `src/game/ruleEngine/field.ts` へ分割済み。
- 次に分割する場合は、次の順で進める。
  - 対象選択
  - ダメージ/撃破処理
  - 継続効果
  - カード個別効果
  - アクション入口
