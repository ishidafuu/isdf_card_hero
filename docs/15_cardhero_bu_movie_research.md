# カードヒーロー部.com対戦動画の調査メモ

## 目的

カードヒーロー部.comの対戦動画ページを、AI改善スレッドへ渡せる外部観戦データとして整理する。

このメモでは動画まわりの取得・構造化・活用候補までを扱う。CPU AI本体、評価関数、ベンチスイート、UIには触れない。

## 生成物

再取得コマンド:

```sh
node scripts/import-cardhero-bu-movies.mjs
```

生成されるファイル:

- `docs/research/cardhero-bu-movies.json`: 対戦動画75件の構造化データ
- `docs/research/cardhero-bu-movies-summary.md`: 件数、マッチアップ、使用カード上位、コメント由来タグのサマリ

取得する情報:

- 対戦動画ページURL、タイトル、投稿日時、最終コメント日時
- ニコニコ動画ID、YouTube分割パートID
- タイトル上のマッチアップ
- ページに掲載された使用デッキ
- デッキコード、エディターURL、前衛/後衛/マジック/スーパーの構成
- コメント件数
- コメント本文から抽出した粗いタグ

取得しない情報:

- 動画ファイル本体
- 動画フレーム、スクリーンショット、字幕
- コメント全文
- AIの教師データとしての局面ラベル

使用デッキはすべて30枚として解析でき、未解決カード名はない。

マッチアップは通常の黒/白対戦が中心だが、`樹聖オリントを使ってみた` のようなテーマ回、低レア注記、トモダチ対戦、グレートマスター表記も混ざる。AI評価に渡す場合は、最初は通常の `黒VS白`、`白VS黒`、`黒VS黒`、`白VS白` に絞るのが扱いやすい。

## データの扱い

`recordedDeck` はページ掲載の使用デッキであり、両者のデッキではない。タイトルの `白VS黒` はマッチアップ情報として扱い、相手デッキを推定しない。

コメントは再配布用データにしない。現状は、将来の手動レビュー候補を探すために `initial_hand_redraw`、`stone_tempo`、`level_up_line`、`citrus_line` などのタグだけを保存する。

動画URLは外部参照として残す。局面データを作る場合も、動画そのものではなく「動画ID、パート、時刻、観察メモ、手動ラベル」を別ファイルに起こす。

## AI改善スレッドへの渡し方

すぐに実装へ入らず、次の順で資料化する。

1. `commentSignalTags` から人間が判断を語っている動画を選ぶ。
2. 該当動画を手動で見て、局面候補を `動画ID / part / 時刻 / 何を比較したいか` の形で記録する。
3. 局面候補を、既存AIの `diff:ai` や投稿デッキベンチに近いテーマへ分類する。
4. AI改善スレッド側で、分類済み局面をテストやベンチに落とすか判断する。

最初の分類候補:

| テーマ | 目的 | 候補タグ |
| --- | --- | --- |
| 初期手札/引き直し | 開幕事故、前衛/後衛の初手価値を見る | `initial_hand_redraw` |
| 石テンポ | ストーン温存、ロストーン、ヤミー、ソウルチャージの価値を見る | `stone_tempo` |
| レベルアップ筋 | 撃破、レベルアップ、スーパー化の準備を捨てないか見る | `level_up_line`, `super_line` |
| シトラス/どこでも系 | 配置とレベルアップ補助の扱いを見る | `citrus_line`, `magic_timing` |
| 終盤の詰め | 盤面価値より勝ち切りを優先できるか見る | `lethal_or_closeout` |

## 優先レビュー候補

最初に見るなら、コメントタグが具体的で、デッキも通常Pro 8なしに近いものを優先する。

| 動画 | 理由 |
| --- | --- |
| No.0001 `白VS黒` | 初期手札、配置、シトラス、レベルアップが同時にタグ化されている |
| No.0032 `白VS白` | 初期手札、石テンポ、シトラス、レベルアップ、マジックタイミングが重なる |
| No.0017 `白VS白` | 石テンポ、配置、シトラス、マジック、終盤寄りタグが重なる |
| No.0021 `妖姫ヨシノを使ってみた？` | 黒マスター側の石テンポ、マジック、スーパー系の確認に使える |
| No.0075 `黒VS黒` | シリーズ終盤で、コメントタグは少ないが石テンポとシトラスの言及がある |
