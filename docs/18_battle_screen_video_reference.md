# 対戦動画由来のバトル画面表示メモ

## 目的

カードヒーロー部.comの対戦動画を、バトル中の盤面表示を改善するための視覚参考として整理する。

ここでは動画の画像や公式UI素材を保存・複製しない。参考にするのは、盤面情報の置き方、視線誘導、密度、操作導線だけに限定する。

## 確認結果

No.0001 `フリー対戦Pro8なし『白VS黒』` のYouTubeページで映像フレームを確認できた。

- カードヒーロー部.comの詳細ページにはYouTubeの古いFlash埋め込みURLが残っている。
- 現在のブラウザでは、詳細ページ内のFlash objectは映像表示としては扱いにくい。
- `docs/research/cardhero-bu-movies.json` の `videos.youtube[].url` を開けば、YouTube側で映像フレームを確認できる。
- 確認した動画はDSキャプチャ由来の低解像度映像で、盤面と手札の情報密度を観察する用途に向く。

## 真似する要素

### 1. 上下でプレイヤー領域を分ける

動画では、相手側と自分側の状態が上下に分かれている。これにより、どちらのターンか、どちらのカードかが直感的に分かる。

現行UIへ取り入れるなら:

- CPU領域を上側、Player領域を下側に固定する。
- マスターHP、ストーン、山札、手札枚数を各領域の近くに寄せる。
- 盤面中央だけを見ても、上下どちらの持ち物か分かるようにする。

### 2. HP/ストーンを小アイコン列で見せる

動画ではHPやリソースが数字だけでなく、横並びの小アイコンとして読める。数値の増減が視覚的に追いやすい。

現行UIへ取り入れるなら:

- `MasterResourceDisplay` のHP/Stone表示を、数字中心からアイコン列中心へ寄せる。
- 10を超える場合だけ `+n` や小さな数値表示を併用する。
- HPとStoneの色・形を明確に分ける。

### 3. 盤面を最優先の視覚中心にする

動画では手札や情報欄より、盤面のカード位置が最初に目に入る。Card Heroでは前衛/後衛/マスター位置が判断の中心なので、この優先順位は現行UIにも合う。

現行UIへ取り入れるなら:

- `.board` を画面中央の主役にする。
- `.side-panel` と `.info-panel` は補助情報に下げる。
- 選択中カードの説明は大きすぎる常設パネルではなく、盤面横のコンパクトな詳細枠に寄せる。

### 4. 手札はカード列として圧縮する

動画では手札が下部に並び、盤面とは別の帯として認識できる。常に全カード詳細を出すより、カード名・種類・コスト・小アイコンを素早く読めることが重要。

現行UIへ取り入れるなら:

- `.hand-list` は横スクロールまたは折り返し1段を基本にする。
- 手札カードは「アイコン、名前、コスト、役割」だけを初期表示する。
- 詳細な技説明は選択時の詳細枠へ移す。

### 5. ターン終了などの主操作を盤面横に固定する

動画では `ターンエンド` が盤面右側に大きく表示され、次に押す操作が分かりやすい。

現行UIへ取り入れるなら:

- ターン終了、選択キャンセル、決定などの主要操作を `.board` 右側の固定操作枠に置く。
- ログやAI Labのような分析機能とは視覚的に分離する。
- CPU vs CPU観戦時は、再生/一時停止/次イベントを同じ枠に置く。

## 真似しない要素

- 公式動画の画面画像、フレーム、ボタン画像、ドット素材は保存しない。
- DS画面の配色や枠線をピクセル単位で再現しない。
- 低解像度映像のノイズ、黒帯、キャプチャ由来の歪みは取り入れない。
- 公式UIの文言や素材をそのまま増やすのではなく、現行プロトタイプの用語に合わせる。

## 現行UIとの接続点

現行の主な接続先:

| 目的 | 現行要素 |
| --- | --- |
| 盤面中央化 | `.battle-area`, `.battle-primary`, `.board` |
| マスター情報 | `.master`, `MasterResourceDisplay` |
| スロット表示 | `.board-slot`, `BoardSlot` |
| 手札帯 | `.hand-area`, `.hand-list`, `.hand-card` |
| 選択中詳細 | `.side-context-panel`, `.board-card-detail`, `CardDetail` |
| ログ/分析補助 | `.side-panel`, `.log-panel`, `.info-panel` |

最初の改修は、構造を壊さずCSSと配置だけで進めるのがよい。ルール処理、AI判断、カードデータには触れない。

## レイアウト案

### Desktop

```text
[上部ツールバー: seed / mode / AI / master / new game]

┌──────────────────────────────────────────────────────────┐
│ CPU HP / Stone / Deck / Hand                             │
├──────────────────────────────────────────────────────────┤
│ CPU後衛L      [不可]       CPU後衛R                      │
│ CPU前衛L      CPU Master   CPU前衛R                      │
│ Player前衛L   PlayerMaster Player前衛R                   │
│ Player後衛L   [不可]       Player後衛R                   │
├───────────────────────┬──────────────────────────────────┤
│ Player HP / Stone     │ 主要操作: End / Cancel / Confirm │
├───────────────────────┴──────────────────────────────────┤
│ Hand: [card] [card] [card] [card] [card]                 │
└──────────────────────────────────────────────────────────┘

右または下: 選択中カード詳細、ログ、AI分析
```

### Mobile

```text
[ターン/状態]
[CPU HP / Stone]
[盤面]
[Player HP / Stone]
[主要操作]
[手札横スクロール]
[詳細/ログはタブ]
```

Mobileでは、ログとInfoを常時表示しない。盤面、操作、手札を最優先にする。

## 実装順

1. `docs`だけで画面方針を固める。
2. `.battle-primary` 周辺だけを対象に、盤面中央化のCSS案を作る。
3. `MasterResourceDisplay` をアイコン列中心に寄せる。
4. `.hand-card` を圧縮し、詳細説明を選択時表示へ寄せる。
5. 主要操作枠を盤面右側または下側に固定する。
6. Playwrightでdesktop/mobileのスクリーンショットを確認する。

## 完了条件

- 盤面、HP/Stone、手札、ターン終了の位置関係が一目で分かる。
- 手札やログが盤面より目立ちすぎない。
- 既存のドラッグ、クリック、ターゲット強調が壊れない。
- 公式動画の画像や素材を直接使っていない。
- AI改善やルール変更と同じコミットに混ぜない。
