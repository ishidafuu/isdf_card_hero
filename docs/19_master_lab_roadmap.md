# Master Lab ロードマップ

## 目的

新マスター案は、既存の `white` / `black` に直接追加せず、まず実験レイヤーで試す。狙いは次の3つ。

- 既存ルール、CPU AI、デッキ検証の本線と衝突しにくくする
- 「ホワイトを壊しすぎない」「ブラックと同じ速攻に寄せない」を検証可能にする
- 候補ごとに勝率、長期戦、特技使用、問題盤面を比較できるようにする

## 基本方針

Master Lab は、本採用前の候補を扱う場所とする。

- 実験マスターIDは `white` / `black` の `MasterId` にはまだ入れない
- 既存魔法で表現できる特技は `magic_ref` として参照する
- 新規効果が必要な特技だけ `experimental_effect` として切る
- 本採用前に、シナリオテストと自動対戦レポートを両方通す

## フェーズ

### Phase 0: 設計台帳

候補マスターの意図、特技、リスク、受け入れ条件をコード上の台帳に置く。

- 成果物: `src/game/masterLab.ts`
- 検証: 候補ID、参照カード、コスト、評価マトリクスの静的チェック
- 現在の対象: デコイマスター、犠牲型、タイミング型
- 状態: 完了

### Phase 1: 既存魔法参照

既存魔法をマスター特技候補として扱える形にする。

- `card_097` 挑発
- `card_128` スケープゴート
- 特技側コストはカード本体と別に上書き可能にする

- 成果物: `listMasterLabActionOptions` / `playMasterLabAction`
- 実装方針: 仮想手札カードを一時追加して既存魔法効果を再利用し、解決後に仮想カードを取り除く
- コスト: カード本体のコストではなく、Master Lab側の特技コストを支払う
- 状態: 完了

### Phase 2: 実験ランナー

実験マスターを `white` / `black` と同じ自動対戦検証に載せる。

現在の次ステップはこのフェーズ。まずはデコイマスターだけを対象にする。

実装は段階的に行う。

1. 1手評価: 実験特技だけを列挙、適用、評価する
2. 混合評価: 既存CPU候補と実験特技候補を同じレポートで比較する
3. 自動対戦: 座席ごとに実験マスターを指定して複数戦を回す

- 新マスター vs ホワイト
- ホワイト vs 新マスター
- 新マスター vs ブラック
- ブラック vs 新マスター
- 新マスター同士

最初のゲートは各組み合わせ100戦とし、次を記録する。

- 勝率
- 平均ターン数、最大ターン数
- failure / warning
- マスター特技の使用回数
- 長期戦になったseed
- 特定特技がロック化していないか

### Phase 3: CPU AI 評価

実験特技をCPUが使えるように、既存の合法手列挙と評価関数へ実験分岐を足す。

優先順位は次の通り。

1. スケープゴート: ブラックの本体詰めを防ぐ場面で使えるか
2. 挑発: 主力を守るために囮へ攻撃を逸らせるか
3. 供養: 行動前限定で相手のレベルアップ機会を消せるか
4. 鎮魂: ブラックの撃破圧を反撃価値へ変えられるか

### Phase 4: シナリオテスト

勝率だけでは壊れ方を見落とすため、小さい盤面テストを作る。

デコイマスターでは、最低限次を確認する。

- バーサクでマスターを詰める攻撃をスケープゴートで受ける
- バーサクで主力を倒しに来る攻撃を挑発で囮へ逸らす
- 挑発された側が、攻撃不能なら通常の選択肢を失わない
- ホワイトの育成陣形を位置破壊しない
- 挑発とスケープゴートだけで永続ロックにならない

### Phase 5: 本採用判定

候補を本採用する条件は次の通り。

- ホワイト、ブラック双方への100戦検証で failure 0
- warning は原因が説明でき、再現seedが残っている
- 極端な相性差がない
- マスターの勝ち方が既存2種と明確に違う
- 実装差分が `masterLab` から本線へ切り出せる

## 初期候補

### デコイマスター

スタンス: 攻撃誘導型。

- マスターアタック: 3コ
- 挑発: 3コ
- スケープゴート: 2コ

狙いは、相手の攻撃を止めるのではなく、当たり先をずらして損な攻撃に変えること。ホワイトのシールド防御とは違い、被害そのものは発生する。ブラックのバーサク速攻には、本体詰めをスケープゴートで受け、主力撃破を挑発で逸らす形で対抗する。

主なリスクは、挑発がロック化することと、スケープゴートがホワイトの防御と被りすぎること。

### 犠牲型

スタンス: 被害変換型。

- マスターアタック: 3コ
- 供養: 2コ、登場済み・未行動の自分モンスター1体を倒して1枚引く
- 鎮魂: 3コ、次の自分ターン開始まで、自分モンスターが相手攻撃で倒れたら相手マスターに1ダメージ

狙いは、ブラックの撃破圧をそのまま受けつつ、倒されることを価値に変えること。供養は相手のレベルアップ機会を直接消すため、行動前限定を基本にする。

### タイミング型

スタンス: 強い瞬間をずらす型。

- マスターアタック: 3コ
- ステイ: 2コ、相手の準備中モンスター1体の登場を1ターン遅らせる
- パワーダウン: 2コ、登場済みモンスター1体の次の攻撃パワーを-1

狙いは、ホワイトの陣形を破壊せず、ブラックの撃破ラインを1ターンずらすこと。

## 実装順

1. `masterLab` 台帳と静的テストを追加する
2. レポート出力スクリプトを追加する
3. デコイマスターだけ実験ランナーに接続する
4. スケープゴートのシナリオテストを追加する
5. 挑発のシナリオテストを追加する
6. 100戦マトリクスを回して結果を `docs/master_lab/results/` に残す
7. 問題がなければ犠牲型、タイミング型へ展開する

## 判定メモ

デコイマスターは第1候補として扱う。理由は、ホワイトの盤面維持ともブラックの速攻とも違う勝ち方になりやすく、既存魔法の挑発・スケープゴートで最小実装から試せるため。
