# スペシャルカード導入仕様

## 目的

通常カード126枚の通常戦を維持したまま、スペシャルカードを追加できるようにする。

スペシャルカードはカードヒーローらしさを上げる追加要素として導入するが、既存のランダムデッキ、公式一致テスト、オートプレイ検証の基準を崩さない。

この文書でいうスペシャルカードは、公式カードリスト上の `スーパー` 種別24枚を指す。実装上のプール名は `special` とし、UIでは `Special` / `スーパー` と表示する。

## 基本方針

採用する。

ただし、デフォルトの通常戦には混ぜない。

- 通常カード126枚は、これまで通り標準カードプールとして扱う。
- スペシャルカードは別カードプールとして扱う。
- ランダムデッキ生成は、初期状態では通常カードのみを使う。
- スペシャルカードは、Deck Setupで明示的に許可した場合だけ固定デッキへ投入できる。
- スペシャルカードは手札から直接召喚できない。
- 対応モンスターがレベルアップ条件を満たした時、手札の対応スーパーカードで変身できる。
- CPUも、デッキに入っているスペシャルカードは通常カードと同じルールで使用する。
- ホワイト以外のマスター導入とは独立して進める。

## 非目標

この仕様では次を扱わない。

- 戦闘後報酬
- 肉システム
- ローグライト進行
- 公式AI完全再現
- スペシャルカード限定の入手条件、売買、希少度運用
- ホワイト以外のマスター実装

## カードプール

カード定義にはプール区分を持たせる。

```ts
type CardPool = "normal" | "special";
```

既存カードはすべて `normal` とする。

公式 `スーパー` カードは `special` とする。

カードIDは公式No.に対応する `card_###` を使う。通常カードで未使用だったNo.を埋める形にし、既存IDとは衝突させない。

公式データを取り込む前に、スペシャルカード一覧を次の情報で固定する。

| 項目 | 必須 |
| --- | --- |
| 公式名 | 必須 |
| 種別 | モンスター / マジック |
| 前衛/後衛 | モンスターの場合は必須 |
| Lv別HP/技 | モンスターの場合は必須 |
| コスト/対象/効果文 | マジックの場合は必須 |
| 公式効果文 | 必須 |
| 実装上の挙動メモ | 必須 |
| 仮アイコン | 任意。ただし表示崩れを避けるためプレースホルダは必須 |

未確認の効果を推測で実装しない。

## デッキルール

デッキの基本ルールは通常カードと同じ。

- 30枚固定
- 同名3枚まで
- 前衛12枚以上
- 後衛6枚以上
- マジック6枚以上

スペシャルカードに追加の投入枚数制限は設けない。

ただし、公式ルール上でスペシャルカード固有のデッキ制限が確認できた場合は、その時点で仕様書と検証を更新してから実装する。

スーパーカードはデッキ30枚には入るが、前衛12枚、後衛6枚、魔法6枚の最低枚数には数えない。

## ランダムデッキ生成

初期状態のランダムデッキ生成ではスペシャルカードを使わない。

理由:

- 通常カード126枚のオートプレイ検証結果を維持するため。
- スペシャルカードの強さや特殊効果でCPU検証が揺れることを避けるため。
- 通常戦の再現性と、スペシャルありの検証結果を分けるため。

将来的にスペシャル入りランダムを使う場合は、Deck Setupに次を追加する。

- `スペシャルカードをランダムデッキに含める`
- スペシャルカードの最大投入枚数
- seed固定時に同じスペシャル入りデッキを再現する検証

初回導入では、このランダム混入は実装しない。

## 固定デッキ編集

Deck Setupに `スペシャルカードを許可` を追加する。

OFF:

- カード選択UIにスペシャルカードを表示しない。
- テキスト直接編集でスペシャルカード名またはIDが入っていた場合は、デッキ検証エラーにする。
- 既存の通常カード126枚だけでデッキを構築する。

ON:

- カード選択UIでスペシャルカードを表示する。
- テキスト直接編集でスペシャルカード名またはIDを解決できる。
- デッキ検証では通常カードと同じ枚数ルールを適用する。

Player / CPUで許可状態は分ける。

例:

- Playerだけスペシャルあり
- CPUだけスペシャルあり
- 両者スペシャルあり
- 両者スペシャルなし

## カード一覧UI

Card Listにはプールフィルタを追加する。

- All
- Normal
- Special

初期表示は `Normal` とする。

スペシャルカードには、通常カードと区別できる表示を付ける。

- `Special` ラベル
- カード詳細のプール表示
- 仮アイコンまたはプレースホルダ

ただし、盤面や手札上ではカードタイプ、HP、技、コスト、効果を優先し、Specialラベルは過度に目立たせない。

## ルール実装

スペシャルカードも、通常カードと同じルール入口を使う。

- `summonMonster`
- `attackWithCommand`
- `playMagic`
- `getCommandTargets`
- `getMagicTargets`
- `runCpuStep`

スペシャルカード専用の攻撃/魔法発動入口は作らない。ただし、登場方法は通常召喚ではなくレベルアップ解決時のスーパー化として扱う。

カード固有効果が必要な場合は、通常カードと同じ効果分岐に追加する。

実装時に守ること:

- 表示される効果だけが発火する。
- 公式効果文、カード詳細表示、実装挙動、テスト期待値を同じコミットで更新する。
- 未実装のスペシャルカードをカードプールに混ぜない。
- 実装途中のカードには `implemented: false` を残さず、投入対象から除外する。

## CPU

CPUは、スペシャルカードを特別扱いしない。

基本方針:

- デッキに入っているカードは、通常カードと同じ候補生成/評価で使う。
- 既存の評価で扱える攻撃、回復、強化、防御、移動、手札/山札選択はそのまま使う。
- 新しい効果種別が出た場合は、CPU評価の簡略化を許さず、最低1つ代表ケーステストを追加する。

CPUの完成条件は公式AI再現ではなく、検証相手として破綻しないこと。

## テスト

スペシャルカード導入時は、通常カードと別に検証を分ける。

必須テスト:

1. カードプールテスト
   - 通常カード126枚が維持される。
   - スペシャルカードが `special` として分類される。
   - デフォルトの `getAllCardDefs` 相当の通常用途で、スペシャルカードが混入しない。

2. デッキ検証テスト
   - スペシャルOFFでは固定デッキに入れられない。
   - スペシャルONでは固定デッキに入れられる。
   - 30枚、同名3枚、前衛/後衛/魔法の枚数制限が効く。

3. UIテスト
   - Deck Setupの許可トグルでカード選択候補が変わる。
   - Card ListでNormal/Specialを切り替えられる。
   - カード詳細にSpecialラベルと効果が表示される。

4. 公式一致テスト
   - 追加したスペシャルカードごとに、最低1つ公式効果に対応する状態変化テストを持つ。
   - ランダム効果がある場合はseed固定で期待値を固定する。
   - 複数対象や手札選択がある場合は手動UI/CPU補完の両方を確認する。

5. オートプレイ検証
   - 通常カードのみの100戦検証は引き続きPASSする。
   - スペシャルONの固定デッキまたは専用プリセットで100戦検証を追加する。

## 導入順

### Step 1: 仕様とデータ境界

状態: 完了。

- この仕様書を追加する。
- `CardPool` の設計を入れる。
- 通常カード126枚を `normal` として固定する。
- スペシャルカードはまだ実カードとして投入しない。

### Step 2: UI/デッキ境界

状態: 完了。

- Deck Setupに `スペシャルカードを許可` を追加する。
- Card Listにプールフィルタを追加する。
- スペシャルOFFでテキスト直接編集のスペシャルカード投入をエラーにする。

### Step 3: 公式スペシャルカードデータ取り込み

状態: 完了。

- 公式名、効果文、種別、ステータス、技、コストを固定する。
- 仮アイコンを用意する。
- 公式 `スーパー` 24枚を `pool: "special"` として投入する。
- `evolvesFrom` で対応する進化元を固定する。

### Step 4: 効果実装

状態: 実装済み。継続してカード単位の公式一致テストを増やす。

- 公式効果文と表示を先に固定する。
- 1カードずつ効果実装と公式一致テストを追加する。
- 通常カード126枚の既存テストを同時に通す。

### Step 5: CPU/オートプレイ検証

状態: 一部完了。

- CPUがスペシャルカードを使う代表ケースを追加する。
- 通常カードのみの100戦検証は継続する。
- スペシャルONの固定デッキ100戦検証は未実施。
- warning seedが出た場合は、通常カード側とスペシャル側を分けて分類する。

## 完了条件

- 通常カード126枚のみの通常戦検証が、導入前と同じ基準でPASSする。
- スペシャルOFFでは、通常戦のカードプール、デッキ生成、固定デッキ検証、Card Listが通常カードのみになる。
- スペシャルONでは、固定デッキにスペシャルカードを入れてプレイできる。
- 追加したスペシャルカードの公式効果、表示、実装、テストがカード単位で一致している。
- スペシャルONの100戦オートプレイ検証で、例外、進行不能、未解決プロンプトがない。

## 未決事項

実装前に確認する。

- スペシャルON専用オートプレイ検証の固定デッキ/seed範囲
- スーパーカードをCPUが積極採用するためのデッキプリセット
