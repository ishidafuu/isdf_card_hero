# 戦闘プロトタイプ仕様

## 目的

カードヒーローの通常バトルを参考に、最小限の戦闘プロトタイプを作る。

この仕様書の目的は、実装前に「何をカードヒーローらしさとして採用するか」を
固定すること。完全再現ではなく、最初に遊べる戦闘を作るための仕様に絞る。

## 仕様記述の原則

プロトタイプではスコープを小さくする。
ただし、採用したルールを曖昧にして実装で補完しない。

- 採用するルールは、入力、状態変化、例外、UI上の選択肢まで決める。
- 公式再現しないルールは、`簡略化` または `未採用` として明記する。
- `未確認` のルールは実装しない。必要なら先に調査して、この仕様書へ反映する。
- 実装中に判断が必要になった場合、コードで先に決めず仕様書を更新する。

## 参照するルール

プロトタイプは、DS版『高速カードバトル カードヒーロー』の通常バトルを主な参照元にする。

公式電子説明書から確認した要点:

- 勝利条件は、相手マスターのHPを先になくすこと。
- 初期手札は5枚。配り直しは一度だけ可能。
- 自分のターン開始時に、ストーン3つとカード1枚を得る。
- 後衛にモンスターがいて前衛が空いている場合、ターン開始時に自動で前衛へ移動する。
- モンスターは前衛2体、後衛2体まで呼べる。
- モンスターを呼ぶと、カードとストーン1個がその場所に置かれ、準備中になる。
- 準備中のモンスターは、次の自分のターンに登場する。
- 登場済みモンスターは1ターンに1回、攻撃か移動のどちらかを行える。
- マスターはシールドを持ち、2パワーまでのダメージを無効化する。
- 相手モンスターを倒したとき、ストーンが残っていれば攻撃したモンスターをレベルアップできる。
- レベルアップではHPや攻撃力が上がり、HPは最大まで回復する。
- ターン終了時に手札が6枚ある場合、1枚捨てる必要がある。
- 自分のターンで命令を出さなかったモンスターは、ターン終了時に気合いだめする。

## 今回採用する範囲

### 採用する

- マスターHPを削る勝利条件
- 前後 x 左中右の2 x 3自陣フィールド
- 前衛/後衛
- 召喚時の準備中
- 次の自ターン開始時の登場
- ターン開始時のストーン3個獲得
- ターン開始時の1ドロー
- 後衛から前衛への自動移動
- 通常モンスターは1体につき1ターン1回の攻撃または移動
- ストーンを使った召喚、マジック、レベルアップ
- ホワイトマスターのアタック、ウェイクアップ、シールド
- マスターの2パワーシールド
- モンスター撃破によるレベルアップ
- 気合いだめ
- 実カード名とカードデータを使った30枚ランダムデッキ
- 取り込み済みカードの複数コマンド、特技コスト、一部性格

### 簡略化する

ここに書いた項目は、曖昧なまま省略するのではなく、プロトタイプ仕様として
別ルールに固定する。

- 初期手札の配り直しは実装しない。
- ターン終了時に手札が6枚以上ある場合、5枚になるまで手札配列の先頭のカードを自動で捨てる。
- 攻撃範囲は、スーパーカードを除く採用カードで必要な範囲タグを実装する。
- レベルアップは、可能な場合はプレイヤーが選択、CPUは自動で実行する。
- デッキ構築はなく、30枚ランダムデッキで開始する。
- マスターカードの種類は1種類に固定する。
- スーパーカードへの進化は行わず、元カードの通常レベルだけを扱う。
- カード個別効果は、1対象UIで扱える即時効果として簡略実装する。

### 未採用

ここに書いた項目は、このプロトタイプではコードにもUIにも出さない。

- スーパーカード
- マスターごとの特殊能力
- 公式と完全一致する複数対象選択、ランダム選択、継続効果UI
- 対人バランス調整
- 制限ターン経過時のHP判定
- 準備中カードへの通常攻撃、移動、入れ替え

## 用語

- `マスター`: プレイヤー本体。HPが0になると敗北する。
- `ホワイトマスター`: Phase 1で使う固定マスター。マスターアタック、ウェイクアップ、シールドを持つ。
- `ストーン`: 召喚、マジック、レベルアップ、特技に使うリソース。
- `モンスター`: 盤面に置かれる戦闘ユニット。
- `準備中`: 召喚直後の状態。まだ行動できない。
- `登場済み`: 準備中が解け、行動できる状態。
- `前衛`: 相手に近い列。
- `後衛`: 自分の前列の後ろにある左右枠。中央は進入不可。
- `気合いだめ`: 未行動でターン終了したモンスターに付く強化状態。

## フィールド

各プレイヤーは、前列/後列 x 左/中/右の2 x 3自陣フィールドを持つ。

- 前列中央はマスター位置。
- 後列中央は空きではなく、進入不可マス。
- モンスター枠は、前列左右と後列左右の4枠。
- マスターはモンスター枠ではない。

```text
相手後列L  [進入不可]    相手後列R
相手前列L  [相手マスター] 相手前列R
自分前列L  [自分マスター] 自分前列R
自分後列L  [進入不可]    自分後列R
```

この図では、上が相手側、下が自分側。
それぞれの前列中央にマスターがいて、後列中央にはカードもモンスターも置けない。

座標は次のように扱う。

- `owner`: `player` または `enemy`
- `row`: `front` または `back`
- `lane`: モンスター枠では `left` または `right`。中央はマスターまたは進入不可マスで、モンスター枠ではない。

モンスターは自陣の前列左右/後列左右の空き枠に召喚できる。
相手陣地には召喚できない。
前列中央、後列中央、盤面外には召喚できない。

### 距離計算

プロトタイプでは、8つのモンスター枠、2つのマスター位置、2つの進入不可マスを次の座標で扱う。

| 位置 | x | y |
| --- | ---: | ---: |
| 相手後列L | 0 | 0 |
| 相手後列中央（進入不可） | 1 | 0 |
| 相手後列R | 2 | 0 |
| 相手前列L | 0 | 1 |
| 相手マスター | 1 | 1 |
| 相手前列R | 2 | 1 |
| 自分前列L | 0 | 2 |
| 自分マスター | 1 | 2 |
| 自分前列R | 2 | 2 |
| 自分後列L | 0 | 3 |
| 自分後列中央（進入不可） | 1 | 3 |
| 自分後列R | 2 | 3 |

モンスター枠同士の `adjacent` 距離は、`abs(x1 - x2) + abs(y1 - y2)` で計算する。
`one_skip` 以上の射程距離、およびモンスターからマスターへの射程距離は、`max(abs(x1 - x2), abs(y1 - y2))` で計算する。
マスター位置は座標上に存在し、相手マスター座標が攻撃射程に入る場合は相手マスターも攻撃対象にできる。
自分マスターは攻撃対象にできない。

## プレイヤー状態

各プレイヤーは次を持つ。

- マスターHP
- ストーン
- 山札
- 手札
- 捨て札
- 自陣フィールド

プロトタイプ仕様として、マスターHPは10とする。

理由:

- 公式電子説明書では通常バトルの勝利条件はマスターHPをなくすこととされている。
- 共有会話と調査メモでは、カードヒーローらしさを確認するには短すぎる決着を避けたい。
- 5HPではレベルアップやストーン管理を見る前に決着しやすい。

## バトル開始

### 初期化

- 互いに30枚ランダムデッキを持つ。
- 各デッキは前衛12枚、後衛6枚、マジック6枚を最低保証する。
- 残り6枚は、前衛、後衛、マジックを含む全カードプールからランダムに選ぶ。
- 同名カードは最大3枚までとする。
- プレイヤーとCPUは別々にデッキを生成する。
- バトル開始時、互いの山札をシャッフルする。
- 互いに手札を5枚引く。
- 先攻はプレイヤー固定とする。
- 初期ストーンは0とする。
- 先攻1ターン目は、ターン開始時のストーン3個を得るが、1ドローは行わない。
- 後攻1ターン目は、ターン開始時のストーン3個と1ドローを行う。

### 最初の自ターン開始

先攻であるプレイヤーは、最初の自ターン開始時にストーン3個を得る。
ただし、先攻1ターン目の1ドローは行わない。

後攻であるCPUは、最初の自ターン開始時にストーン3個とカード1枚を得る。

## ターン進行

1. ターン開始
2. 準備中モンスターの登場
3. 後衛から前衛への自動移動
4. ストーン3個獲得
5. 1ドロー
6. 行動
7. ターン終了
8. 手札上限処理
9. 未行動モンスターの気合いだめ

乱数を使う処理は山札シャッフルだけとする。
通常プレイではランダムにシャッフルし、自動テストでは固定seedを使って同じ手札順を再現できるようにする。

### ターン開始

自分の準備中モンスターを登場済みにする。

同じレーンで、自分の後衛に登場済みモンスターがいて前衛が空いている場合、
そのモンスターは前衛に自動移動する。

その後、ストーン3個とカード1枚を得る。

自動移動は左右レーン単位で判定する。

- 自分後衛Lに登場済みモンスターがいて、自分前衛Lが空なら、自分前衛Lへ移動する。
- 自分後衛Rに登場済みモンスターがいて、自分前衛Rが空なら、自分前衛Rへ移動する。
- 同じレーンの前衛が埋まっている場合、その後衛モンスターは移動しない。
- 準備中から登場済みになったモンスターも、この自動移動の対象になる。

### 行動

行動フェーズでは、次を任意の順序で実行できる。

- モンスターカードを使って召喚する
- マジックカードを使う
- マスターHPを1使って1ドローする
- ホワイトマスターの特技を使う
- 登場済みモンスターのコマンドを選んで攻撃させる
- 登場済みモンスターを移動させる
- 登場済みモンスターにためるを行わせる
- ターン終了する

通常の登場済みモンスターは、1ターンに1回だけ攻撃、移動、ためるのいずれかを行える。
カード個別仕様に `2行動` などの例外がある場合は、そのカードの行動上限を使う。

### マスターHPドロー

自分の行動フェーズ中、マスターHPを1使うことでカードを1枚引ける。

このHP消費はマスターHP減少として扱う。
そのため、基本ルールに従って、HPを1使ったプレイヤーはストーンを1得る。

使用回数に上限はない。
マスターHPが0になるまで、同じターン中に何回でも使える。

マスターHPが1のときにも使える。
その場合、HP消費でマスターHPが0になり、その時点で敗北する。

山札が0枚のとき、マスターHPドローは使えない。
この場合、HPコストも山札切れペナルティも発生しない。

### ホワイトマスターの特技

Phase 1では、互いにホワイトマスターを使う。
ホワイトマスターは手札を使わず、ストーンを消費して次の特技を使える。

| 特技 | コスト | 対象 | 効果 |
| --- | ---: | --- | --- |
| マスターアタック | 3 | 相手前列左右の登場済みモンスター1体 | 2Pで攻撃する。マスターは対象にできない。 |
| ウェイクアップ | 2 | 自分または相手の準備中モンスター1体 | 対象を登場済みにする。 |
| シールド | 2 | 味方の登場済みモンスター1体 | 対象が受けるパワーを1減らす。効果は対象の持ち主の次ターン開始時まで。 |

マスターアタックでモンスターを倒しても、レベルアップ権は発生しない。

ウェイクアップで自分の準備中モンスターを登場済みにした場合、そのモンスターは同じターン中に行動できる。
ウェイクアップで相手の準備中モンスターを登場済みにした場合、そのモンスターは攻撃やマジックの対象にできる。
ウェイクアップでは後列から前列への自動移動は発生しない。

シールドは重ねがけできない。
シールドは攻撃やマジックなどのパワーを1減らす。
ボムゾウの自爆反動のような固定ダメージは、シールドで減らせない。

## 召喚

モンスターカードは、自陣の空き枠に召喚できる。

召喚手順:

1. 召喚先の空き枠を選ぶ。
2. 手札からモンスターカードを選ぶ。
3. ストーン1個を消費する。
4. 選んだ枠に準備中カードとして置く。

準備中のモンスターは、次の自ターン開始時に登場済みになる。

準備中のモンスターは次の対象にならない。
相手の準備中カードは、登場するまでカード名、HP、レベル、コマンドを非公開にする。
ログでも相手の準備中召喚カード名は表示しない。

- 攻撃
- 移動
- レベルアップ
- 気合いだめ

準備中カードを登場させる特殊効果や、準備中カードへの干渉は実装しない。

## 移動

登場済みモンスターは、行動として移動できる。

許可する移動:

- 自陣の空き枠へ移動
- 自陣の登場済みモンスターとの位置入れ替え

移動したモンスターは、そのターン行動済みになる。
気合いだめ状態で移動した場合、気合いだめは解除される。
準備中モンスターとは入れ替えられない。

## 攻撃

モンスターは、カードに定義された攻撃方法で対象を選ぶ。

プロトタイプで使う攻撃範囲:

- `adjacent`: 登場済みモンスターのうち、距離1のモンスターを攻撃する。相手マスター座標が隣接している場合は相手マスターも攻撃できる。
- `one_skip`: 登場済みモンスターのうち、1マス飛ばした距離2のモンスターを攻撃する。左右レーンやマスター列はまたいでよい。相手マスター座標が射程2にある場合は相手マスターも攻撃できる。
- `any_monster`: 登場済みモンスター1体を距離に関係なく攻撃する。
- `any_target`: 登場済みモンスター1体、または相手マスターを距離に関係なく攻撃する。
- `master`: 相手マスターを攻撃する。

公式電子説明書では、基本はとなりのマスのみ攻撃可能で、特殊マークにより離れたマスにも届く。
このプロトタイプでは、カードごとに上記の範囲タグを割り当てる。

攻撃範囲はブロッカーを見ない。
たとえば `one_skip` は、1マス飛ばした距離2なら、同じレーン、斜め先、左右前衛同士でも対象にできる。
ブロッカーや貫通の公式再現は未採用とする。

射程はマンハッタン距離ではなく、3列 x 4段の盤面座標で、横方向と縦方向のうち大きいほうの距離で扱う。
射程2なら1マス飛ばした距離2の左右レーンへ届く。
将来、射程3の攻撃を採用する場合は、2マス飛ばした距離3の左右レーンへ届くため、
自分後列右から相手後列左にも届く。

攻撃は味方の登場済みモンスターも対象にできる。
ただし、攻撃したモンスター自身は対象にできない。
味方モンスターを倒した場合、攻撃側マスターのHPを1減らす。
このHP減少でも基本ルールどおり、減ったHPぶんのストーンを得る。
味方を倒した場合、レベルアップ権は発生しない。

### マスターへの攻撃

相手マスター座標が攻撃射程に入っている場合、相手マスターを対象にできる。
`any_target` と `master` は、距離に関係なく相手マスターを対象にできる。
`any_monster` はマスターを対象にできない。

マジックカードが明示的に相手マスターを対象にできる場合も、マスターシールドを適用する。

マスターには2パワーまでのダメージが通らない。

ダメージ計算:

```text
実ダメージ = max(0, 攻撃パワー - 2)
```

例:

- 2Pで攻撃: 0ダメージ
- 3Pで攻撃: 1ダメージ
- 4Pで攻撃: 2ダメージ

### 攻撃パワー補正

モンスターコマンドの攻撃パワーは、次の順で補正する。

1. カードに定義された基礎パワーを取る。
2. 攻撃側が気合いだめ中で、使うコマンドが現在レベルの上段技なら+1する。
   気合いだめ中にモンスターコマンドを使った場合、上段技/下段技に関係なく気合いだめを解除する。
3. 攻撃側にパワーアップが付いていれば+1し、パワーアップを解除する。

上段技はカードデータ上の `commands[0]` とする。
下段技には、射程が1でも気合いだめの攻撃補正は乗らない。
パワーアップはモンスターコマンドだけに適用し、マジックカードには適用しない。

## ダメージ

モンスターへのダメージは、HPを直接減らす。
気合いだめ中のモンスターがダメージを受ける場合、受けるダメージを1減らし、
気合いだめを解除する。最低ダメージは0。

HPが0以下になったモンスターは倒れる。
倒れたモンスターはフィールドから取り除かれ、持ち主の捨て札に置かれる。

カード個別仕様で明記した反動を除き、攻撃したモンスター自身への反動、反撃、同時撃破は未採用とし、発生しない。

## ストーン

### 増える条件

公式電子説明書では、ストーンは次の条件で増える。

- 自分のターン開始時
- マスターのHPが減ったとき
- モンスターが倒されたとき

プロトタイプでは次の仕様に固定する。

- ターン開始時: 自分のストーンを3増やす。
- 自分のマスターHPが減ったとき: シールドなどを適用した後の実HP減少量と同じ数だけ、自分のストーンを増やす。
- 自分のモンスターが倒されたとき: そのモンスターに置かれていたストーンを自分に戻す。

マスターHP減少によるストーン獲得は基本ルールとして扱う。
攻撃、マジック、山札切れペナルティなど、原因に関係なくマスターHPが実際に減った場合は、
減ったHPと同じ数だけそのプレイヤーのストーンを増やす。

モンスターに置かれているストーンは `invested_stones` として管理する。

- 召喚時、モンスターの `invested_stones` は1になる。
- 1レベルアップするたびに、`invested_stones` に1を加える。
- モンスターが倒れたとき、持ち主は `invested_stones` と同じ数のストーンを得る。
- マジック使用や特技で消費したストーンは戻らない。

例: 召喚後に2レベルアップしたLv3モンスターは、召喚時の1個とレベルアップ分の2個で
`invested_stones` が3になる。そのモンスターが倒れた場合、持ち主にストーン3個が戻る。

### 消費する条件

- モンスター召喚: 1個
- マジック使用: カードごとのコスト
- レベルアップ: 1レベルにつき1個
- 特技: カードごとのコスト

## レベルアップ

攻撃したモンスターが相手モンスターを倒した場合、レベルアップ権を得る。

ルール:

- モンスターの攻撃で敵モンスターを倒した場合だけ、攻撃したモンスターがレベルアップ権を得る。
- 味方モンスターを倒した場合は、レベルアップ権ではなく味方撃破ペナルティを解決する。
- マジックで敵モンスターを倒しても、レベルアップ権は発生しない。
- 倒した相手モンスターのレベル数まで上げられる。
- 1レベル上げるごとにストーン1個を消費する。
- 自分のストーンが足りない場合、その分は上げられない。
- モンスターごとの上限レベルを超えて上げられない。
- レベルアップするとHPが最大まで回復する。

レベルアップ可能数は次で決める。

```text
可能数 = min(倒した相手モンスターのレベル, 自分のストーン, 攻撃したモンスターの上限レベル - 現在レベル)
```

プレイヤーがレベルアップ権を得た場合、UIでは `0` から `可能数` までのレベルアップ数を選ぶ。
`0` を選ぶとキャンセルになり、ストーン、レベル、HPは変化しない。
`1` 以上を選ぶと、選んだ数だけストーンを消費し、同じ数だけレベルを上げ、最後にHPを最大まで回復する。

例: Lv1の味方モンスターがLv2の相手モンスターを倒した場合、上限レベルとストーンが足りていれば
2レベル分上げられる。上限がLv3なら、Lv1からLv3まで上げられる。

処理順:

1. ダメージを解決する。
2. 倒れたモンスターを捨て札に置く。
3. 倒れたモンスターの `invested_stones` を持ち主に戻す。
4. 攻撃側がレベルアップ数を選ぶ。
5. レベルアップする場合、攻撃側のストーンを消費し、攻撃したモンスターを強化する。

CPUは `可能数` が1以上なら、常に最大数まで自動でレベルアップする。

## 気合いだめ

登場済みモンスターは、行動として `ためる` を選ぶことで気合いだめできる。
`ためる` は攻撃や移動と同じく1回分の行動として数える。

ターン終了時、そのターンに命令を出さなかった登場済みモンスターも自動で気合いだめする。
これは通常の1回行動モンスターが何もせずターン終了した場合のための処理であり、
すでに攻撃、移動、ためるを1回以上行ったモンスターには発生しない。

2回行動できるモンスターは、1回攻撃または移動した後でも、残り行動で `ためる` を選べる。
同様に、先に `ためる` を選び、まだ行動回数が残っている場合は、同じターン中に気合いだめ状態で行動できる。
すでに気合いだめ中のモンスターは、重ねて `ためる` ことはできない。

気合いだめの効果:

- 次に上段技で攻撃するとき、その技のパワーを1増やす。
- 次に攻撃を受けるとき、受けるダメージを1減らす。

上段技/下段技に関係なく、気合いだめ中のモンスターが攻撃したら気合いだめは解除される。
下段技を使った場合、攻撃補正は発生しないが、行動したため気合いだめは解除される。
気合いだめ中のモンスターを移動させた場合も、行動したため解除される。
被ダメージ軽減が発動した場合も解除される。
複数回行動できるモンスターでも、1回以上攻撃、移動、ためるを行っていれば、自動では気合いだめしない。

## 手札上限

ターン終了時に手札が6枚以上ある場合、5枚になるまで捨てる。

自動で最も古い手札から順に捨てる。
手札廃棄UIは後で追加する。

## 山札切れ

ターン開始時のドローなど、強制ドロー処理で山札が0枚の場合、カードを引けない。
その代わり、ドローしようとしたプレイヤーのマスターHPを1減らす。

これは山札切れ専用の追加ストーン処理ではない。
マスターHPが1減るため、基本ルールに従って、そのプレイヤーはストーンを1得る。

山札切れペナルティによってマスターHPが0になった場合、そのプレイヤーは敗北する。
捨て札を山札に戻す処理は行わない。

## CPU

CPUは検証用の決定的AIとする。
ランダム選択は行わず、同じ盤面なら同じ手を選ぶ。

CPUは毎回、合法手を列挙し、1手先の盤面評価で最も高い手を選ぶ。
v1.1で列挙する行動は次に限定する。

- モンスター攻撃
- マスターアタック
- モンスター召喚
- ためる
- ターン終了

評価では、相手マスター撃破、敵モンスター撃破、レベルアップ可能な撃破、役割に合った召喚、
有効攻撃がない場合のためるを高く見る。
味方モンスターへの攻撃と0ダメージ攻撃は、CPUの候補から除外する。

CPUは可能ならレベルアップする。
CPUは手札廃棄を自動で行う。
CPUはマスターアタックを使う。
CPUはマスターHPドロー、マジックカード、ウェイクアップ、シールド、任意移動を使わない。
同点の場合は、勝利、敵撃破、召喚、非致死攻撃、ためる、ターン終了、フィールド表示順、手札順の順で固定的に選ぶ。

## 操作UI

クリック操作は、これまで通りカード、コマンド、対象の順に選んだ時点で実行する。

ドラッグ&ドロップ操作は即実行しない。
ドロップ元とドロップ先から実行可能な行動が1つ以上見つかった場合、その行動を保留中の選択として表示し、
プレイヤーが `確定` ボタンを押したときに実行する。
`キャンセル` を押した場合、保留中の選択を破棄する。

ドラッグ&ドロップで採用する候補:

- 手札モンスター -> 自陣の空きマス: 召喚
- 手札マジック -> 有効対象: マジック使用
- 登場済み自分モンスター -> 自陣の空きマスまたは登場済み自分モンスター: Move / Swap
- 登場済み自分モンスター -> 攻撃可能対象: アタック

登場済み自分モンスターを味方マスへドロップした場合、アタック可能でも Move / Swap を優先して選択する。
同じドロップ先に複数の攻撃コマンドが届く場合、カードデータ上の上段技から順に最初に届くコマンドを選択する。

## 表示演出

CPUターンは一括で即時解決せず、UI上では約0.5秒間隔で1手ずつ進める。
これにより、召喚、攻撃、ためる、ターン終了の流れをログと盤面で追えるようにする。

盤面上で状態が変わったマスやマスターは、短いハイライトを出す。
ログは、攻撃/ダメージ、召喚、レベルアップ、気合いだめ、支援、ターン進行の種類ごとに色分けする。

## 実装時の状態モデル

- `GameState`: 戦闘全体、ターン数、現在の手番、勝敗
- `PlayerState`: HP、ストーン、山札、手札、捨て札、フィールド
- `SlotState`: 所有者、前衛/後衛、左右レーン、モンスター有無
- `MonsterState`: カードID、HP、レベル、準備中/登場済み、行動回数、行動上限、気合いだめ、次攻撃強化、シールド
- `CardDef`: 種別、コスト、レベル別ステータス、コマンド、効果
- `CommandDef`: コマンド名、パワー、範囲タグ、ストーンコスト、反動
- `Action`: 召喚、モンスターコマンド、移動、マジック、マスターHPドロー、マスター特技、ターン終了

## カード個別仕様

カードデータはカードヒーロー部のカードリスト（https://www.cardhero-bu.com/card/）を参照する。
スーパーカードは今回の移植対象から除外する。
前衛46枚、後衛26枚、マジック54枚の合計126枚をカードプールとして取り込む。

カードアイコンは、テスト用の仮画像としてカードヒーロー部の32pxアイコンを `public/card-icons/` に保存して使う。
後で自作画像に差し替える前提とする。

ゲーム開始時のデッキは、126枚のカードプールから30枚をランダム生成する。
これは正式なデッキ構築ではなく、特殊挙動を含むカードを広く引けるようにする検証用デッキである。

### ランダムデッキ

- デッキ枚数は30枚固定とする。
- 同名カードは最大3枚までとする。
- 前衛カードを12枚、後衛カードを6枚、マジックカードを6枚、最低保証枠として選ぶ。
- 残り6枚は全カードプールからランダムに選ぶ。
- 残り6枚によって、前衛、後衛、マジックの最終枚数は最低保証数を上回ることがある。
- プレイヤーとCPUは同じ生成ルールを使うが、別々のランダムデッキを持つ。

### 性格・特殊処理

- ボムゾウの `自爆`: この通常攻撃を使った場合、対象へのダメージ、撃破、レベルアップ選択を解決した後、ボムゾウ自身も1ダメージを受ける。この反動は固定1ダメージで、気合いだめやパワーアップの影響を受けない。
- ポリスピナーの `2行動`: 1ターン中に2回まで攻撃または移動できる。
- モーガンの `アークドライブ`: `any_target` として扱い、敵の登場済みモンスターまたは相手マスターを対象にできる。
- 0P攻撃: 対象選択は可能だが、ダメージは0として処理する。
- パワーアップ: 同じ対象への重ねがけはできない。強化対象が攻撃するか、現在の手番プレイヤーのターンが終了したら解除する。
- スーパーカード化: このプロトタイプでは未採用。Lv2からスーパーへ進化できるカードも、通常レベル上限までで止める。
- カード個別効果: スーパーカードを除く採用カードは対象選択と効果処理を持つ。複数対象やランダム選択を要求する効果は、プロトタイプ用に1対象または決定的な簡略効果として処理する。

## 完了条件

仕様上の最小プロトタイプは、次を満たしたら完了とする。

- ローカルで1戦を最後まで遊べる。
- 初期手札5枚、通常ターン開始時1ドロー、先攻1ターン目ドローなしが動く。
- ストーン3個獲得、召喚1個消費が動く。
- マスターHPを1使って1ドローでき、HP1でも使用でき、山札0枚では使用できない。
- 山札切れ時にドローできず、マスターHPが1減る。
- ターン終了時に手札が6枚以上なら、5枚になるまで古い順に捨てる。
- モンスターを準備中で置き、次の自ターンに登場する。
- 前衛/後衛2体ずつの配置がある。
- 前衛が空いたとき、後衛モンスターがターン開始時に前進する。
- 登場済みモンスターが攻撃または移動できる。
- ホワイトマスターのマスターアタック、ウェイクアップ、シールドが動く。
- モンスター撃破と捨て札移動が動く。
- レベルアップとHP全回復が動く。
- マスターの2Pシールドが動く。
- `any_target` 攻撃とサンダーで相手マスターにダメージを与えられる。
- マスターHPが0になったら勝敗が決まる。
- ランダム30枚デッキが同名3枚制限、前衛12枚、後衛6枚、マジック6枚の最低保証を満たす。
- CPUが最低限ターンを進められる。
