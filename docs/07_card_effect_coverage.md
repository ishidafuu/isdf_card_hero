# カード特殊効果の網羅テスト基盤

## 目的

カード効果の実装漏れを、個別のバグ報告だけでなくテスト基盤側でも検出できるようにする。

このドキュメントでは、現在のテストが担保している効果群と、まだ簡略化または未対応として扱う効果を明示する。

コード上の一覧は `src/game/cardEffectCoverage.ts` に置き、`tests/game/cardEffectCoverage.test.ts` でカードデータとの差分を検出する。

## カバー対象

| 領域 | 対象 | 主な検証 |
| --- | --- | --- |
| カードプール | 通常カード126枚、仮アイコン、未実装フラグ | カード投入数、カード種別、アイコン、未実装フラグなし |
| 対象選択 | 全マジック、全モンスター技 | 盤面上で少なくとも1つ合法対象を持つ |
| 特殊射程 | 射程2、射程3、貫通、直線、特殊射程 | ヤンバル、ピグミィ、ケントゥリアス、グングニエル、ロロ系 |
| リソース | コスト、山札切れ、HPドロー、ストーン、レベルアップ | HP減少時ストーン、山札切れペナルティ、投資ストーン返却 |
| 複数対象 | 二重対象、入れ替え、手札選択、山札カテゴリ選択 | 二重の盾、ワープ、シフトチェンジ、リフレッシュ、カードサーチ |
| ランダム | ランダムレベル、ランダムストーン、ランダム追加ダメージ | seed固定で再現可能な期待値 |
| 継続防御 | シールド、特殊防御、浄化時の残存/解除 | 女神の加護、竜の盾、スケープゴート、デスチェーン |
| 行動制限 | 特技封じ、挑発、石化/呪い、ダークホール | 行動候補とターン処理への反映 |
| 撃破/被ダメージ | 復活、反撃、退却、献身、怨念、仮死 | ダメージ処理と撃破処理の割り込み |
| CPU利用 | マジック、シールド、ウェイクアップ、移動後攻撃 | 判断理由ログと代表ケース |
| seed検証 | 100戦単位のオートプレイ | 例外、未解決プロンプト、長期戦、不審判断のartifact化 |

## テスト基盤

追加した `tests/game/cardEffectCoverage.test.ts` は、次を検出する。

- カードデータに新しいマジックカテゴリが増えたのに、カバー一覧へ追加されていない。
- モンスター技に新しい特殊traitが増えたのに、カバー一覧へ追加されていない。
- カバー領域に検証パスが紐付いていない。
- 簡略化/未対応一覧に、現在挙動、理由、次作業が書かれていない。

既存の `tests/game/rules.test.ts` は、具体的なルール挙動を担保する。今回の新規テストは、それらを横断するメタテストとして扱う。

## 未対応/簡略化カード効果一覧

| ID | 状態 | 内容 | 現在の扱い | 次作業 |
| --- | --- | --- | --- | --- |
| `super_cards_pending` | 未対応 | スペシャルカード | 通常カード126枚のみを投入 | 導入時にデータ、UI、CPU、専用テストを追加 |
| `per_card_assertion_gap` | 未対応 | 全カード個別の公式一致テスト | 効果群代表テストと全カード合法対象テストで検出 | 特殊効果カードから1枚ずつ期待値テストを増やす |
| `multi_target_defaulting` | 簡略化 | 複数対象カード | `secondaryTarget`未指定時に決定的補完 | 手動UIで追加対象選択、CPUで評価付き選択 |
| `hand_choice_defaulting` | 簡略化 | 手札/山札選択カード | 選択値未指定時に決定的補完 | 手動UIの選択パネルとCPU評価を追加 |
| `random_resolution_seeded` | 簡略化 | ランダム効果 | `GameState.randomSeed`で決定的解決 | 結果演出と乱数結果ログを追加 |
| `cpu_magic_heuristic` | 簡略化 | CPU特殊効果判断 | 浅い評価で判断 | warning seedを使って移動往復、複数対象、手札選択評価を改善 |
| `temporary_original_icons` | 簡略化 | 攻略サイト由来の仮アイコン | テスト用画像として表示 | 自作画像へ差し替え、命名規則とチェックを追加 |

## 運用

カード効果を追加または修正した場合は、次を同じコミットで更新する。

1. `src/game/cardData.ts`
2. `src/game/rules.ts` または `src/game/cpuAi.ts`
3. 具体挙動のテスト
4. `src/game/cardEffectCoverage.ts`
5. 必要ならこのドキュメント

warning seedが出た場合は、まず `docs/06_warning_seed_review.md` と同じ粒度で分類する。カード効果漏れならルールテストへ、CPU判断品質ならCPUテストまたはオートプレイ検証へ戻す。
