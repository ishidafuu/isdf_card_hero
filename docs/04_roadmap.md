# 今後のロードマップ

## 目的

このロードマップは、現在の戦闘プロトタイプを土台に、検証可能な順番で機能を積み上げるための作業順を定義する。

最優先は、カード追加やローグライト化ではなく、1戦の戦闘が安定して何度も検証できる状態を維持すること。

## 現在地

完了済み:

- Vite + React + TypeScript の戦闘プロトタイプ基盤
- 2 x 3盤面、マスター位置、進入不可マス
- ランダム30枚デッキと同名3枚制限
- 召喚、準備中、登場、自動前進
- ストーン、マスターHPドロー、山札切れペナルティ
- モンスター攻撃、マスターアタック、マジック、ためる
- 味方攻撃と味方撃破ペナルティ
- 撃破時レベルアップ、投資ストーン返却
- ドラッグ&ドロップによる召喚、攻撃、移動/入れ替えの選択
- 手札、配置カード、マスター枠の情報表示
- ログの種別表示、盤面上のダメージ/KO演出
- 両者CPUオートプレイと行動ウェイト設定
- 独立オートプレイ検証コマンド
- 失敗/長期戦/不審判断seedのartifact保存
- review-400-499 artifactのwarning seedレビュー
- CPU AI v1.3
  - モンスター攻撃
  - マスターアタック
  - マジック
  - モンスター召喚
  - ウェイクアップ
  - シールド
  - 任意移動 / 入れ替え
  - 移動/入れ替え後攻撃の浅い評価
  - ためる
  - ターン終了
  - 判断理由ログ
- 100戦オートプレイ検証
- カード特殊効果の網羅テスト基盤
- 未対応/簡略化カード効果一覧

## 優先順位

### Phase 1: 戦闘プロトタイプの安定化

目的:

現在の1戦プロトタイプを、手動プレイとCPU相手の反復検証に耐える状態にする。

状態:

完了。引き続き不具合が出た場合は、このPhaseへ戻って修正する。

完了済み:

- CPU選択理由のログ表示
- ターン中に何が起きたかを追いやすくするログ分類と表示改善
- 操作可能対象、攻撃可能対象、選択中カードの視認性改善
- 勝敗後のリスタート導線
- 主要ルールの回帰テスト追加
- 100戦オートプレイ検証ログ

### Phase 2: CPU AIの検証基盤強化

目的:

AIの強さを上げる前に、例外、進行不能、極端な長期戦、ルール破綻をseed単位で追える状態にする。

状態:

完了。今後は失敗seedが出たときにartifactをレビューして、CPU評価またはルール実装へ戻す。

完了済み:

- オートプレイ検証のseed範囲を拡張する
- 失敗seedの再現手順をログ化する
- 長期戦になったゲームの終盤ログを保存できるようにする
- CPU判断理由と実際の盤面変化を突き合わせるレビュー手順を作る
- `npm run validate:auto-play -- --seed-start 400 --count 100` の独立コマンド化
- artifactに完全な `GameState`、盤面要約、ログ末尾、直近判断履歴を保存
- review-400-499のwarning seedをレビューし、長期戦は進行不能ではなく、移動/入れ替えの多発はCPU判断品質の課題として分類

完了条件:

- 100戦単位の検証を任意のseed範囲で再実行できる。
- 失敗時にseed、turn、直近ログ、未解決状態が確認できる。
- CPU判断の改善前後を同じseedで比較できる。

### Phase 3: CPU AIの品質改善

目的:

CPUが合法手を選べる状態から、意図に近い手を選べる状態へ進める。

実装対象:

- マジック使用判断の評価調整: 完了
- ウェイクアップ後に同ターンで攻撃できる場合の優先度調整: 完了
- シールド対象の危険度評価の精度向上: 完了
- 移動後攻撃、入れ替え後攻撃の浅い探索: 完了
- 味方攻撃、自己犠牲、レベルアップ優先の例外ケース検討

完了条件:

- CPUが有効攻撃につながるウェイクアップを安定して選べる。
- CPUが高価値モンスターを守るシールドを過不足なく選べる。
- CPUが後衛を後列へ戻す、前衛を前列へ出す移動を選べる。
- 移動だけでターンを浪費し続けない。

### Phase 4: カードセット検証

目的:

移植済みカードの特殊効果が、戦闘ルールとCPU判断の両方で破綻していないか確認する。

状態:

進行中。効果群ごとの代表テスト、全カード/全技の合法対象テスト、全カード/全技の解決スモークテスト、カバー一覧のメタテストは追加済み。次は公式挙動と完全一致する個別期待値を増やす。

完了済み:

- 攻撃特殊効果ごとの代表テスト追加
- 継続効果、死亡時効果、複数対象、ランダム対象の代表ケース追加
- 全マジック/全モンスター技の解決スモークテスト追加
- カード定義とルール実装のカバー一覧追加
- 未対応/簡略化しているカード効果の明示
- warning seedに出た同一ターン内の移動/入れ替え往復抑制
- 複数対象/手札選択UIの明示化
- デッキ内容、捨て札、手札破棄UI
- CPUの複数対象/手札/山札選択評価

実装対象:

- 全カード個別の公式一致テスト追加
- CPUの特殊効果評価の長期価値改善

完了条件:

- 各カードの主要効果に最低1つ以上の回帰テストがある。
- 攻略サイト由来データと実装差分がドキュメントで追える。
- 未実装の効果がUI上またはドキュメント上で曖昧に残っていない。

### Phase 5: プレイアビリティ改善

目的:

検証しやすさを維持しつつ、手動プレイ時の迷いを減らす。

実装対象:

- 山札、捨て札、除外相当の確認UI
- 手札廃棄UI
- ログフィルタ
- 行動履歴の詳細表示
- デッキ内容の確認UI

完了条件:

- カード効果と現在状態を画面上で確認できる。
- 不正操作の理由が、ボタンまたは選択中パネルから確認できる。
- 直近の行動結果がログを追わなくても理解できる。

### Phase 6: 戦闘後報酬と肉システム

目的:

カードヒーロー風戦闘から、このプロジェクト固有の面白さへ進む。

実装対象:

- 戦闘結果画面
- 撃破報酬
- 肉の獲得
- 肉による変異
- 変異候補の提示

着手前に確定する仕様:

- 肉はモンスター単位で持つか、プレイヤー全体の報酬にするか
- 変異はカードを書き換えるか、ラン中の個体だけを書き換えるか
- 死亡、変異、成長をどの程度永続化するか

完了条件:

- 1戦後に報酬選択が発生する。
- 肉を選ぶことで、次の戦闘に明確な変化が出る。
- 変異結果がプレイヤーに理解できる。

### Phase 7: ローグライト最小ループ

目的:

単発戦闘から、複数戦をつなぐ最小ループへ進む。

実装対象:

- ラン開始
- 戦闘選択
- 報酬選択
- 次戦への持ち越し
- 簡易ボス戦
- ラン終了

完了条件:

- 3戦から5戦程度の短いランを最後まで遊べる。
- 戦闘後の選択が次戦に影響する。
- 敗北またはクリア後に、すぐ次のランを開始できる。

## 常時守る運用

- 採用するルールは、実装前に仕様書へ明記する。
- 未確認ルールは推測で実装しない。
- 仕様変更をしたら、実装とテストを同じタイミングで更新する。
- 作業完了時は、検証を通してからコミットする。
