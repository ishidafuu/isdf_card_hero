# 戦闘後報酬と肉システム仕様

## 目的

Phase 6では、単発戦闘の後に次戦へつながる選択を追加する。

この段階ではローグライト全体の永続化やショップは入れず、1戦勝利後に報酬を1つ選び、次の戦闘デッキへ反映されるところまでを最小仕様とする。

## 採用仕様

- ラン開始時、プレイヤーは通常カード126枚から作られる30枚固定デッキを持つ。
- 戦闘ごとに、ラン中のプレイヤーデッキをシャッフルして使う。
- CPUデッキは従来通り戦闘ごとにランダム生成する。
- プレイヤーが勝利した場合、報酬候補を3つ提示する。
- 報酬を1つ選ぶと、ラン中のプレイヤーデッキを更新して次の戦闘へ進む。
- デッキは常に30枚固定、同名カード3枚まで、前衛12枚以上、後衛6枚以上、魔法6枚以上を維持する。

## 撃破報酬

- 戦闘中に倒れたモンスターを `GameState.defeatedMonsters` に記録する。
- 報酬候補では、プレイヤーが撃破したCPUモンスターを優先して参照する。
- 撃破カードがない場合は、通常モンスターから決定的seedで候補を補完する。

## カード報酬

- カード報酬は「獲得カード」と「入れ替え対象カード」を必ずセットで表示する。
- 獲得カードは、同カテゴリのカードと入れ替える。
- 同カテゴリとは、前衛、後衛、魔法の3分類。
- 同名3枚制限に達しているカードは、報酬候補から除外する。

## 肉報酬

- 肉は現時点では在庫化せず、勝利報酬として即時に変異を起こす。
- 肉報酬は「どのモンスターの肉か」「デッキ内のどのカードが何へ変異するか」を必ず表示する。
- 肉の元になったモンスターと同じ前衛/後衛カテゴリ内で変異する。
- 変異後も30枚固定、同名3枚制限、カテゴリ最低枚数を維持する。

## 未採用

- 肉の所持数管理
- 肉を複数個消費する変異
- 変異によるカード能力値の直接書き換え
- 敗北後のラン継続
- スペシャルカード報酬
- 3戦から5戦のラン終了条件

これらはPhase 7以降で扱う。

## 完了条件

- 勝利後に報酬選択画面が出る。
- カード報酬または肉報酬を選ぶと、次戦のプレイヤーデッキが変化する。
- 報酬適用後も30枚固定とカード投入枚数ルールが崩れない。
- 報酬候補と適用結果がテストで固定されている。
