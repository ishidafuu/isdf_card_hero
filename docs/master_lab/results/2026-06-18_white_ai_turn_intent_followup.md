# White AI Turn Intent Follow-up

生成: 2026-06-18 JST

## 背景

ひいら氏ブログの「メインフェーズは2つのフェーズ」では、ターン中の行動を大きく次の2種類に分けて考える整理がある。

- このターンの仕事: そのターン中に成果を出す行動。
- 次のターンへの布石: 次の自分ターンを強くする準備行動。

既存の調査メモでも、この観点は HBR-001 として未着手のCPU AIレビュー項目に入っている。

- `docs/16_hiirach_blog_research.md`: 「メインフェーズは2つのフェーズ」
- `docs/research/hiirach-blog-review-queue.md`: HBR-001

今回の白AI調整では、勝率上は `attack_monster` 系の補正が一部有効に見えたが、白の本質である石管理、レベルアップ回復、小打点調整、複数行動ユニットの目的化までは十分に見えていない。次の改善ループでは、この2分類を白AIの評価内訳として使う。

## 白AIへの変換

白マスターは守りに石を使えるが、守りすぎると石が枯渇する。したがって、各候補手を単純な総合点ではなく、まず以下に分ける。

### このターンの仕事

- 敵モンスターを倒す。
- 自分のモンスターをレベルアップさせる。
- 負傷したモンスターをレベルアップさせ、HP回復を得る。
- 相手マスターへリーサルまたは重要な本体打点を通す。
- 相手のレベルアップ餌、バーサク打点源、次ターンの危険な攻撃元を減らす。
- 被リーサルを防ぐ。

この分類では、石を使ってもよい。ただし、非リーサルのマスターアタックや、撃破にも反撃にもつながらないシールドは成果不足として扱う。

### 次のターンへの布石

- 次ターンに4体で行動できる盤面を残す。
- 守ったモンスターが次ターンに攻撃、撃破、レベルアップへ変換できる。
- ウェイクアップ後の露出リスクを負っても、次ターンの手数や配置が改善する。
- ピグミィなどの小打点で敵HPを調整し、次の撃破/レベルアップを作る。
- ポリスピナーの1回目の行動が、2回目の撃破、退避、位置調整、本体打点へつながる。
- 次ターンに必要な `shield` / `wake_up` / レベルアップ用の石を残す。

布石行動では、石を使うほど評価基準を厳しくする。特に `shield` は「守った結果、次に何が起きるか」を見ずに撃たない。

## 診断ログとして見る項目

次ループでは勝率だけでなく、次の分類を記録する。

| 指標 | 見たいこと |
| --- | --- |
| `executionActionRate` | ターン中の明確な成果行動がどれだけ選ばれたか。 |
| `setupActionRate` | 次ターンへの布石行動がどれだけ選ばれたか。 |
| `lowStoneAfterSetup` | 布石後に石が1以下になった回数。守りすぎ検出。 |
| `shieldConvertedNextTurn` | シールド対象が次ターンに攻撃/撃破/レベルアップした回数。 |
| `shieldNoConversion` | シールドしたが次ターン成果に変わらなかった回数。 |
| `woundedLevelUpHeal` | 負傷モンスターのレベルアップ回復が発生した回数。 |
| `pygmySetupHit` | ピグミィの小打点が撃破圏作りやレベルアップ筋に寄与した回数。 |
| `polyspinnerPurposefulFirstAction` | ポリスピナーの1回目行動が2回目の成果へつながった回数。 |
| `polyspinnerWasteAction` | ポリスピナーが2行動を雑に消費した疑いのある回数。 |

## 実装方針

いきなり白プロファイルの重みを変えない。まずは評価内訳とログ分類を作り、前回の `pressure_white_baseline` と `pressure_attack_monster8_shield4` を比較する。

1. `MasterLabDecisionEvent` か白AI専用レポートに、候補手の `execution/setup` 分類を追加する。
2. シールドとウェイクアップは、選択時点の理由だけでなく、次の自分ターンで成果に変換されたかを追跡する。
3. ピグミィとポリスピナーはカード固有ログとして扱う。最初から強い補正を入れず、まず雑な使用率を測る。
4. ログ分類後、必要なら白専用の局面評価として `stoneConservation`, `woundedLevelUpRecovery`, `smallDamageSetup`, `multiActionPurpose` を追加する。

## 次ループ案

候補は少なく、診断を厚くする。

- `pressure_white_baseline`
- `pressure_attack_monster8_shield4`
- `pressure_attack_monster_plus4` 相当の弱補正版
- `stone_guard_no_proactive_shield`: 石温存を強め、成果の薄いシールドを抑える診断候補
- `wounded_levelup_setup`: 負傷レベルアップ回復と、その布石シールドだけを少し強める診断候補

母数は `games-per-matchup 8-10` から始める。勝率差より、石枯渇、シールド変換率、ポリスピナー/ピグミィの目的化を優先して読む。

## 期待する判断

白AIの改善は「守るAI」ではなく、次の形を目標にする。

- このターン必要な仕事はきちんと終わらせる。
- 次ターンの布石は置くが、石を枯らさない。
- シールドは守った後の成果が見える時だけ使う。
- レベルアップ回復を前提に、負傷モンスターを価値ある駒として扱う。
- ピグミィやポリスピナーの細かい手数を、目的のある打点調整として使う。
